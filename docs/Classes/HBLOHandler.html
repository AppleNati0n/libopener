Overview

A handler is a class that is consulted by Opener when the system is about to open a URL. If a handler returns an NSURL from openURL:sender:, this URL will be used instead. If the handler doesn’t want to replace the URL, it returns nil.

Opener handles enabling and disabling of the handler for you. If you would like to override this and provide your own preferences, you can set preferencesBundle and preferencesClass.

Identifier of the handler, used internally by Opener.

Discussion

To prevent conflicts, use a reverse DNS name, such as your package’s identifier. If your handler previously supported Opener versions prior to 2.0, use the name you passed to [HBLibOpener registerHandlerWithName:block:].

Discussion

– openURL:sender:

Called when a URL is opened in order to check if your handler has an override of the URL.

- (nullable id)openURL:(NSURL *)url sender:(nullable NSString *)sender

Parameters

url

The original URL that the user wants to open.
sender

The bundle identifier of the app that invoked the URL open request. If the request was invoked through SpringBoard or a non-graphical program such as sbopenurl, this argument is nil.

Return Value

An NSURL that will be opened instead of the original, an NSArray of NSURLs for multiple apps, or nil to not change the URL.
